Xelure Citi Case Study

About the Project

This project was created to provide analyzation and validation against the data. This project is consist of two (2) tabs. "Check File" and "Upload File"

  • Check File: This tab is where you compare the data from Certificate Holder Statements and Enhanced Loan-Level Data.
  • Upload File: This tab is where you upload csv file/s from Enhanced Loan Level Data.

Findings and Notes

I didn't have problem working with Certificate Holders Statements PDF Files, as all were formatted the same. In terms of Enhanced Loan-Level Data Files. The columns names are hard to determine which columns corresponds to PDF File data. I only determined the following:

  • Scheduled Principal
  • Curtailments
  • Prepayment
  • Insurance Fee

I had a hard time to derive the Net Liquidation Proceeds , I noticed this item is the sum of Principal Losses . I used the Principal Losses to counter check the Net Liquidation Proceeds

Check File

For simplicity of this project, I only worked with the page of the PDF files where the "Total Principal Fund Available" is located. Assuming all PDF files have the same format.

This tab will extract the data needed from pdf and will create a table -- I only worked with "SOURCE OF FUNDS" column. It will parse the file date (yy-mm) from the file name and will query in the database all the records with the same file date. Once queried, the records will be accumulated and will also form a table for comparison.

I only uploaded the columns needed for "Principal Funds" into the database.

Upload File

For the purpose of this case study, I only uploaded columns needed for Principal Funds. You can upload multiple files at once. If the file is already uploaded, it will tell the file already exists.
